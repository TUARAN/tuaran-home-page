## /works 页面全面视觉重构

**用户诉求**：可视化太差、太多卡片、缺乏权重层次。

**第一版（被否决）**：权重体系正确但视觉"死板"——所有元素共用 `border-l-[3px]` 左边框模式，缺乏变化。

**最终版设计原则**：
1. **四层权重体系**：L1 Hero 精选（大号装饰数字+展开布局）→ L2 类型导航（紧凑标签条）→ L3 作品分区（三种行样式按优先级分级）→ L4 底部信息（纯文本）
2. **去卡片化**：移除所有 WorkPreviewMark、FeaturedWorkCard、WorkCard、TypeShortcut 卡片组件
3. **多样视觉语言**：
   - MainHero：大号装饰数字 "01" + 超大标题
   - SubHeroCard：上方渐变色条 + 正文
   - HighlightRowNumbered：左侧装饰序号 + 详细信息
   - HighlightRowCompact：底部细线分隔 + 紧凑文字
   - CompactRow：极简文本行
   - 分区头：大号装饰数字重叠 + 标题
4. **优先级驱动布局**：>=85 高优先级用装饰数字大行，75-84 中优先级用紧凑行，<75 低优先级用双列极简网格
5. **统计数字**：从卡片改为大号 serif 字体垂直堆叠

**文件**：`app/(site)/works/page.jsx`（完整重写，从 ~30 张卡片降为 0 张卡片）

## /works 页面第三次视觉重构 —— 打破死板感

**用户反馈**："权重设计没问题，但是页面太难看了，好死板"

**问题诊断**：
- 所有元素共用单调的左边框/底边线模式
- 纯白底+灰色文字，缺乏氛围感和层次
- 排版过于均匀，每个区域占用相同的视觉密度
- 无装饰元素、无纹理、无视觉惊喜

**本轮改进**（文件：`app/(site)/works/page.jsx` 完整重写）：

1. **分区底色化**：每个类型分区用不同的 `accentBg` 底色包裹（Product 暖棕、AI 冷蓝、System 绿、Research 紫、Tool 金），交替显示
2. **5 种分区头变体**：不再千篇一律的左对齐大数字+标题模式，改为：
   - Product → 左对齐 + 下划线 + 右侧描述
   - AI → 垂直色块 + 标题并排
   - System → 超大淡色数字 + 细线分隔
   - Research → 居中 + 菱形装饰分隔
   - Tool → 小圆点 + 同行标题 + 右侧描述
3. **装饰几何元素**：每个分区随机出现淡色大圆（CircleDeco）、对角线（DiagonalLines）增加视觉层次
4. **Hero 渐变背景**：主 Hero 用渐变色洗背景 + 180px 超大隐约数字
5. **次 Hero 卡片化**：带底色边框 + 左上角序号 + 顶部色条
6. **全局色调统一**：所有文字色从零散的 hex 改为统一的 `#15140f → #5E5F55 → #9A9B90 → #C0BDB0` 灰阶体系
7. **类型导航 Pill 化**：从 border-bottom 标签改为 `rounded-full` 药丸形状，底色跟随类型色
8. **链接色**：统一用 `#6E6F64` 中性灰代替固定的 `#8b5a1f` 棕色，hover 变深
