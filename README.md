# 涂阿燃的个人网站

这是 [2aran.com](https://2aran.com) 的源码仓库。

它不是一个博客模板，也不只是作品集。说得直白点，它是一个**前端 + AI Agent 工程师（同时是奶爸、是 founder）的长期个人主场**：把写作、研究、作品、公开工具、对外产品和站内互动，全部放进同一个 Web 应用，按"值得投入 20 年、每日复利"的方式持续迭代。

如果用一句话概括它在干嘛——**用 AI 把内容、作品和增长这三件事连起来的个人操作台**。第一人称的判断与写作是地基，可演进的作品与工具是样本，外接的几个产品和全网分发是变现面。

如果你发现内容错误、页面问题、链接失效，或者有新的栏目和工具建议，欢迎到 [Issues](https://github.com/TUARAN/tuaran-home-page/issues) 提出。

## 这个站的重心

按重要性，而不是按目录罗列：

- **写作与判断（内核）**：原创专栏、随笔日志、AI 协助的公司 / 人物 / 行业 / 工具 / 政策深度调研。区别于"又一个技术博客"的地方，是这里更看重第一人称的长期判断，而不是信息搬运。
- **会迭代的作品与工具（样本）**：可视化富页面（AI Runtime 三极格局、癌症全景、日月运行、Token 用量调研等）、世界杯数据页、股票 / 加密快照分析、skill-center、dad-stack、浏览器端 Web LLM 实验。这些是"把想法做成系统"的活样本，会反复演进。
- **灵感流（高频更新）**：`/feed` 短平快地分享有审美的图片、视频、资源与一句话观点。
- **对外产品与增长（变现面）**：首页直接连向博主联盟、前端周看、AI 分发大师三个在经营的产品，以及全网多平台的内容分发与社交沉淀。
- **站内互动与运营（底座）**：登录、评论、点赞、留言、燃币激励、游客互动，加上仅作者可见的运营后台与自动化采集。

## 功能地图

### 内容与阅读

- 首页信息流：聚合推荐阅读、作品、社交、对外产品与个人入口。
- 文章系统：技术文章、随笔、专题内容，带完整 RSS 输出（精选文章 / 调研全文 / 灵感流）。
- 研究系统：按公司、人物、主题归档深度调研，支持加密内容（明文不进静态产物）。
- 灵感流：`/feed` 图片 / 视频 / 资源 / 观点短内容，独立详情页与 OG 卡片。
- 资源库：外部资料、原文整理、专题页、链接谱系与索引。
- 阅读体验：移动端适配、目录、复制 Markdown、分享入口、中英双语外壳。

### 作品与项目

- 作品集与 AI 项目页：个人项目、agent、模型调度与自动化实践。
- 可视化富页面：长期维护的数据 / 概念可视化专题。
- 服务页：对外展示可提供的技术服务与合作方向。
- 更新日志：记录站点与项目的持续迭代。

### 数据与工具

- 短链与分享系统。
- 公开舆情：定时采集公开新闻与 Hacker News，做轻量聚合（owner 工作台）。
- 世界杯数据页：赛程、分组、积分与榜单。
- 股票 / 加密快照分析：分钟级市场快照与横向分析（owner 工作台）。
- Web LLM 实验：浏览器端（WebGPU）大模型推理与能力验证。

### 账号与互动

- 登录方式：GitHub / Google OAuth，邮箱验证码注册与密码登录。
- 文章评论、点赞与阅读互动；游客评论与登录后合并。
- 燃币体系：签到、评论、解锁等站内激励机制。
- 留言、分享与部分受保护内容访问。

### 管理与自动化

- 仅作者可见的 Admin 后台（AdminPageGate 鉴权）。
- 内容、导航、资源、RSS 订阅、任务与用户运营管理。
- Cloudflare D1 迁移、GitHub Actions 定时采集、Pages Functions / D1 / R2 集成。

## 技术栈

- **框架**：Next.js 15 App Router + React 19（JavaScript，无 TypeScript）
- **样式**：Tailwind CSS 3
- **运行平台**：Cloudflare Pages / Functions（`@cloudflare/next-on-pages`）
- **数据**：Cloudflare D1、R2
- **认证**：自定义 Edge session（GitHub OAuth + 签名 Cookie），支持 Google OAuth 与邮箱验证码 / 密码登录
- **运行时**：静态 / ISR 页面、Cloudflare Edge API、浏览器端 WebGPU 推理三层并存
- **自动化**：GitHub Actions、Wrangler、站内采集任务

> 历史遗留：仓库中仍保留已废弃的 NextAuth 相关文件（`[...nextauth]` 路由返回 410）与本地版 `lib/stompDb.js`，线上均不使用。

## 开发者说明

这个项目可以本地运行，但很多功能依赖作者自己的 Cloudflare、OAuth、邮件和数据库配置。外部开发者如果只是浏览代码，通常不需要完整复刻这些环境。

最小启动流程：

```bash
npm install
cp .env.example .env.local
npm run dev
```

默认本地地址是 [http://localhost:3000](http://localhost:3000)。涉及登录、邮件、D1、R2、采集任务、加密内容的功能，需要按 `.env.example` 补充自己的环境变量。

常用检查：

```bash
npm run build        # 本地构建
npm run pages:build  # Cloudflare 构建（@cloudflare/next-on-pages）
```
