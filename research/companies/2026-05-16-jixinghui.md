## 二、产品与技术

### 2.1 业务模式（基于源码逆向）

从前端 Vue 路由与组件结构可还原：

```
src/
  router/index.js           ← 路由：/ /login /admin /publish /activity/:id /certificate/:id
  pages/
    Index.vue               ← 首页（Header + Task + KOL + Client + Footer + Modal）
    Login.vue               ← 登录
    activity/Publish.vue    ← 活动发布
    activity/Detail.vue     ← 活动详情
    admin/Index.vue         ← 管理后台
    certificate/Certificate.vue   ← 证书页
  config/config.js          ← 后端 API 地址
```

后端 API 端点（来自 `config/config.js`）：

| 端点 | 用途 |
|---|---|
| `/api/custom/info` | 客户（科技企业）信息 |
| `/api/user/login/qrcode` | 扫码登录二维码 |
| `/api/mine` | 当前用户信息 |
| `/api/tasks/list` + `/api/task` | 任务列表 / 详情 |
| `/api/kols/media` | KOL（博主）排行 |
| `/api/cos/getAuthorization` | 腾讯云 COS 凭证 |

**业务模型还原**：

> 极星会 = **"企业发布开发者营销/技术布道任务 ↔ 技术博主接单完成 ↔ 平台发参与证书"**的撮合平台。

具体角色：

| 角色 | 行为 |
|---|---|
| 科技企业（客户） | 注册 → 发布"任务"或"活动"（如：写一篇华为云某产品测评 / 参加某黑客松宣讲） |
| 技术博主（KOL） | 注册 → 浏览任务 → 接单 → 完成 → 拿到分发奖励或证书 |
| 平台 | 撮合 + 任务管理 + 证书签发 + （推断）抽佣 / SaaS 订阅 |

### 2.2 技术栈（源码暴露）

- **前端**：Vue 3 + Vite + ElementPlus + TailwindCSS（tailgrids）+ Prism.js + animate.css + Socket.IO
- **后端**：自有 API（itkol.top），未暴露具体语言
- **服务器**：Nginx + 宝塔面板（路径暴露 `/www/server/pro/gitfund/...`）
- **存储**：腾讯云 COS（appid 1256482441）
- **DNS**：DNSPod（腾讯云）

### 2.3 关键观察 ⚠️

> 🟡 **生产环境直接运行 Vite dev 模式**：网站源代码包含完整的 `@vite/client`、HMR runtime、source map 和未压缩的 Vue 单文件组件源码。
>
> **解读**：要么团队规模小到没有专门的 Web 工程师做 build，要么仍在内测打磨阶段，要么对前端工程化重视度低。**任何一种解读都指向"非常早期"**。

## 八、值得关注的点

### 8.1 业务模式层面

- **赛道选择正确但门槛高**：
  - 正确：IT 博主营销在云厂商 / 大模型公司预算里**确实是一个真实的支出项**
  - 高门槛：客户决策链长（市场部 + 开发者关系部 + 法务）+ 博主侧供给分散 + 平台需要长期沉淀信任
- **与"博主联盟"等社群型组织正面对位**：能否提供**社群所不能提供的"标准化交易 + 工程化撮合"**是核心命题

### 8.2 工程与团队层面 ⚠️

- **生产环境跑 Vite dev 是非常严肃的负面信号**：
  - 安全风险：源码完全暴露，竞争对手可一键扒站
  - 性能风险：页面体积是生产 build 的 5-10 倍
  - 团队成熟度：暗示团队还没到"工程化标配"阶段
- **建议尽调动作**：
  - 如果作为合作 / 投资标的，**首先要核实工程团队是否到位**
  - 如果作为竞品分析，可以**直接下载源码做对照学习**

### 8.3 信息透明度风险

- 注册资本、营收、客户名单、团队成员、融资信息全部未公开
- 公开搜索引擎覆盖极低
- **这种透明度对外部决策非常不友好**：合作 / 投资 / 招聘任何方向都需要深度尽调

### 8.4 个人视角

- **极星会的真实面目** = "一个早期 / 内测状态的 IT 垂类 KOL 任务撮合 SaaS"，不是大厂、不是融资亮眼标的、不是行业头部
- **赛道判断**：
  - 这个赛道**真的有**：云厂商和大模型公司每年都有"开发者营销"预算
  - 这个赛道**真的难**：博主联盟、自营计划、广告代理等多种形态都在抢
  - 极星会的"工程化撮合 + 证书体系"是有差异化空间的，但需要时间证明
- 对外部观察者：
  - **技术博主** → 可以注册看一下，作为额外的接单渠道；不要 all-in
  - **云厂商市场部** → 可以试用作为博主资源池补充，但不要替代主供应商
  - **如果做投资** → 太早期，估值锚点不清，建议跟踪 12-18 个月看 PMF
  - **如果做合作 / 借鉴** → 业务模式 + 完整可读源码是难得的学习样本
- **个人结论**：**"早期 + 真问题 + 工程化弱"** —— 是个值得跟踪的小项目，但目前无法做出强判断。**12 个月后再评估一次**。
