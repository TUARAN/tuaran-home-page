## 一、基本信息

- **集团成立**：1990 年（香港）；另有来源称 1989 年起步，差异可能来自"集团雏形 vs 正式注册"
- **大陆主体**：万达宝软件（深圳）有限公司，**2001-11-02** 注册，法人 **胡宏基**，性质港澳台法人独资
- **总部 / 主要团队**：
  - 集团总部：香港
  - 大陆深圳主体地址：深圳市南山区海德三道天利中央广场 20 楼 06-09 室
  - 办公点：香港、深圳、广州、上海（部分资料还提到新加坡布点）
- **英文品牌**：**Multiable**（multiable.com.cn）
- **创始团队 / 核心人物**：公开资料以法定代表人 **胡宏基** 为代表，集团创始团队未做高调披露
- **融资历史**：**未融资**（36 氪 PitchHub 显示无融资记录）—— 典型自营自给的港资软件公司
- **员工规模**：约 **200–300 位** 专业咨询顾问 + 技术人员（不同来源口径不一）
- **服务客户**：累计 **3000–6000+ 家** 制造 / 贸易企业（来源差异较大，自报口径，需谨慎采信）
- **主营业务**：企业信息化软件（ERP / CRM / HCM / BI / 工作流 / POS）+ 实施咨询 + AI 智能体（LAIDFU）

## 二、产品与技术

### 产品矩阵

| 产品 | 简介 |
|---|---|
| **M18 ERP** | 企业资源规划，核心产品；1998 年即推 C/S 架构，2001 年推出"全国首个纯 Java ERP" |
| **M18 CRM** | 客户关系管理 |
| **M18 HCM** | 人力资本管理 |
| **M18 BI** | 商业智能分析 |
| **M18 POS** | 零售门店系统 |
| **M18 Workflow** | 工作流引擎 |
| **LAIDFU（来福）** | 2025-2026 推出的 AI 智能体平台，对标企业级 Agent |

### 技术路线 / 架构亮点

- **多语言 / 跨地区**：1996 年就支持繁简英三套界面，配合大陆/港澳/台湾不同管理模式 —— 这是它在大湾区华人企业圈深耕的差异化抓手
- **Java 栈**：2001 年纯 Java ERP；老牌 BS 化架构，技术栈相对传统
- **部署**：**云端 + 私有化双轨**，强调"对数据安全和隐私要求较高的企业"私有化能力
- **LAIDFU 技术点**：自然语言处理 + 数据索引算法，跨本地 / 云端 / 企业内部平台一键文档搜索；供应商多维度评估（交货及时性、质量、价格）

### 与竞争对手的差异点

- **港资定位**：相对金蝶 / 用友的内地大盘玩法，万达宝定位在"港澳台 + 大湾区 + 海外华人贸易圈"细分市场，多语言、跨境业务流是它的传统强项
- **行业聚焦**：硬件、进出口贸易、零售、汽配、食品饮料、建筑/HVAC、电子、塑胶、生鲜 —— 偏制造和贸易，避开金蝶/用友的财务税务腹地
- **客单价 / 服务模式**：以"项目实施 + 咨询"为主，不是 SaaS 自助开通模式，更接近传统 ERP 厂商打法

## 三、商业化

- **收入模型**：license 销售 + 实施咨询 + 维护订阅；典型重交付的 ToB 企业软件营收结构
- **关键客户 / 行业落地**：
  - 行业页覆盖：零售、生鲜、电子、塑胶、汽配、进出口贸易、建筑/HVAC、食品饮料、酒店
  - 公开案例多为大湾区中型制造 / 贸易客户，缺少头部品牌大客户的高调露出
- **公开营收 / 估值**：**未披露**。36 氪显示"未融资"，可推断为依靠自有现金流运营的传统软件商
- **报价**：未公开列价，全部走"咨询接洽 + 定制报价"路径，400-800-0084 是官方接入电话

## 四、近期动作（按时间倒序）

- **2025-2026**：推 **LAIDFU（来福）AI 智能体** —— 文档智能搜索 + 供应商评估，定位企业级 Agent
- **2024-2025**：行业资讯页持续更新 ERP / 零售 / 生鲜行业方案文章，做 SEO 内容引流（行业关键词长尾词矩阵）
- **2023**：36 氪 PitchHub 页面仍维护，但无融资公告
- **历史里程碑**：
  - 1990 集团成立（香港）
  - 1996 推出繁/简/英多语言 ERP
  - 1998 区内首批 C/S 架构 ERP 开发商
  - 2001 推出"全国首个纯 Java 企业资源管理系统"，同年深圳主体注册
  - 2003 获评深圳软件开发行业十强企业

## 五、值得关注的点

### 1. AI 切入晚，且产品力存疑

LAIDFU 是 2026 年才进入主推视野的产品，对比金蝶 KIS 智能助理、用友 YonGPT、SAP Joule，万达宝起步晚。官方页面对 LAIDFU 描述模糊（"自然语言处理 + 数据索引算法"），未披露使用哪个底层模型、私有化部署的硬件门槛、Token / 调用价格等关键信息 —— 这种描述方式更像营销层包装，**实际产品深度需要 demo / POC 验证**。

### 2. 私域玩法、信息透明度低，外部尽调困难

- 官网 SPA 渲染，搜索引擎抓取受限，第三方很难拿到结构化产品 / 案例数据
- 没有融资 → 没有外部信披压力 → 客户数 / 员工数 / 营收全部"自报口径"
- 36 氪、百度百科上的数据陈旧，**做对接前建议直接电话 + 邮件索要白皮书 / 客户名单**

### 3. 港澳台 + 大湾区 SMB 制造贸易圈是它的护城河，也是天花板

不同来源对客户数的口径是 3000–6000 之间，但都集中在**大湾区中小制造 / 贸易 / 零售商**。这是金蝶 / 用友覆盖薄弱的细分市场，也是 SAP / Oracle 看不上的客单价。万达宝在这层活了 35 年，说明定位扎实；但同样意味着**很难突破到大型集团客户或非华语市场**，AI 红利的边际增长空间有限。

### 4. 港资软件公司的典型样本

- 自营、不融资、靠咨询实施现金流
- 老板深耕、团队稳定（200-300 人量级 35 年没扩张到 1000+）
- 产品迭代偏稳健，技术栈传统（Java），但跟得上新概念（云、AI Agent）
- 在赛道头部巨头集中度提升的趋势下，**长期可能被金蝶 / 用友云原生方案、或本土 AI Agent SaaS 蚕食**

### 5. 如果你是潜在用户 / 合作方

- 适合：港澳 / 大湾区中等规模制造 / 贸易企业，要求多语言、跨境、私有化部署
- 不适合：追求快速 SaaS 开通、对 AI 智能体能力深度要求高、需要标准化 API 生态的客户
- 决策建议：拿 LAIDFU 做一次实际 POC，对比金蝶云 ERP + 通义千问、用友 YonGPT，看产品差异；不要被"35 年港资老牌"的故事冲昏，**今天的企业软件竞争维度已经换到 AI 原生 + 生态了**
