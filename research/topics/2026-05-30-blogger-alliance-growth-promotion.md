> **调研定位**：这篇不是再次介绍“博主联盟是什么”，而是回答一个更具体的经营问题：**如果现在要推广博主联盟这个业务，找人帮忙推广可以做什么、需要多少钱、怎么评估值不值。**
>
> **业务上下文**：根据博主联盟官网公开信息，当前平台定位是“连接 AI 产品与技术影响力”的技术博主联盟与品牌增长平台，公开展示 **41 位+技术博主、1.4M+程序员/科技粉丝覆盖、300 篇+累计投放推文、660 万+累计曝光、20 个+辐射社群**，服务包覆盖推文、引流、社群、AI Access 联运、出海云访问联运等。

## 一、先定性：博主联盟不是普通广告平台，而是“开发者营销撮合服务”

博主联盟卖的表面是“技术博主推广”，本质是三件事：

1. **技术产品解释能力**：把 AI 产品、开发者工具、云服务、API、开源项目讲成开发者愿意看的内容。
2. **可信第三方分发**：通过真实技术博主而不是品牌自说自话，把产品带进开发者讨论语境。
3. **长期内容资产**：每篇测评、教程、案例都会沉淀为搜索 / AI 搜索 / 社群转发里的可见度。

所以它的获客对象不是“所有需要营销的人”，而是更窄的几类：

| 目标客户 | 典型痛点 | 适合卖点 |
|---|---|---|
| AI 编程工具 / Agent 产品 | 有产品但开发者不知道、不信任、不会用 | 技术测评、教程、真实试用、GEO 占位 |
| 大模型 API / 云厂商 / DevTools | 需要影响开发者和技术决策者 | KOL 矩阵、场景化内容、开发者社群 |
| 开源 / 商业开源项目 | 需要 Star、贡献者、教程、中文社区声量 | GitHub + 掘金/CSDN/知乎内容组合 |
| 出海工具回国推广 | 不懂中文开发者社区，不知道找谁发 | 一站式撮合、翻译/本地化内容、社群触达 |
| 技术课程 / 训练营 / 社群 | 需要可信作者背书与首批用户 | 博主推荐、社群宣发、案例共创 |
| 垂直 B2B SaaS | 泛广告无效，需要解释复杂能力 | 深度内容、白皮书解读、行业 KOL |

**一句话判断**：博主联盟自身推广最关键的不是曝光，而是让上述客户相信：“我找你，比我自己一个个找博主更省事、更稳、更可复盘。”

## 二、推广目标分三层：不要一上来只盯成交

早期平台获客要把目标拆开，否则容易把推广做成“发了一堆内容但不知道有没有用”。

| 层级 | 目标 | 可量化指标 | 合理判断周期 |
|---|---|---|---|
| 认知层 | 让目标客户知道博主联盟 | 品牌词搜索、官网访问、案例页访问、收藏/转发 | 2-4 周 |
| 询盘层 | 让潜在客户愿意加微信/留表单 | 有效咨询数、预算明确咨询数、客户行业分布 | 2-6 周 |
| 成交层 | 形成试单和复购 | 试单金额、成交率、毛利率、复购率 | 1-3 个月 |

当前阶段建议主 KPI 不要定“曝光量”，而定：

> **有效品牌方询盘数 + 单个有效询盘成本 + 首单成交率。**

其中“有效询盘”建议定义为同时满足三项：

1. 有明确产品 / 品牌 / 官网；
2. 有推广目标（曝光、试用、注册、社群、GEO、联运等）；
3. 有预算或可判断预算区间。

只说“了解一下”“怎么合作”的泛咨询可以记录，但不要算核心 KPI。

## 三、找谁帮忙推广：六类人可以用，但结算方式不同

### 3.1 技术 KOL / KOC：适合做信任背书

**能做什么**：

- 写“为什么开发者产品需要博主联盟”观点文；
- 写合作案例拆解；
- 在掘金、公众号、知乎、CSDN、小红书、X 发平台介绍；
- 在自己的技术群里转发招募或品牌合作信息。

**预算参考**：

| 博主类型 | 单篇/单次参考价 | 适合用法 |
|---|---:|---|
| KOC / 千粉到数千粉 | 300-1500 元 | 铺种子内容、测试话术 |
| 中腰部技术博主 | 1500-8000 元 | 可信介绍、案例解读、社群触达 |
| 强垂直 KOL / 十万粉级 | 8000-30000+ 元 | 品牌背书、节点传播、联合活动 |

**评估重点**：

- 不是看总粉丝，而是看“有没有 AI / DevTools / 云服务 / 开发者产品受众”；
- 是否愿意实名背书；
- 是否能讲清楚平台价值，而不是只发硬广；
- 评论区是否有品牌方、创始人、市场/运营/DevRel 人群出现。

### 3.2 商务 BD / 渠道销售：适合直接拿询盘

**能做什么**：

- 定向联系 AI 产品方、开发者工具、云厂商、开源项目商业团队；
- 在创始人社群、AI 产品群、DevRel 圈做转介绍；
- 帮忙约会议、收集需求、推进试单。

**结算方式**：

| 模式 | 参考价格 | 适合阶段 |
|---|---:|---|
| 纯佣金 | 成交额 10%-30% | 有成熟产品包、客单价清楚 |
| 底薪 + 佣金 | 3000-8000 元/月 + 5%-15% | 需要连续跟进客户 |
| 单线索结算 | 100-500 元/有效线索 | 试水渠道质量 |
| 单会议结算 | 300-1000 元/有效会议 | B2B 客单较高时 |

**评估重点**：

- 是否有真实 AI / SaaS / DevTools 客户资源；
- 过往是否做过 B2B 服务销售；
- 是否愿意按 CRM 记录跟进过程；
- 能否接受“有效线索定义”和“撞单规则”。

> **建议**：早期不要轻易给“永久渠道分成”。可以设置 3-6 个月保护期：由该 BD 首次引入并成交的客户，在保护期内复购给佣金，超过保护期重新约定。

### 3.3 技术媒体 / 垂直社区：适合做品牌可信度

**能做什么**：

- 采访主理人：为什么做技术博主联盟；
- 发布案例：某 AI 产品如何通过技术博主做开发者增长；
- 联合做线上直播 / 圆桌：AI 产品怎样做开发者营销；
- 在社区 Banner、公众号、社群做曝光。

**预算参考**：

| 类型 | 参考预算 | 说明 |
|---|---:|---|
| 垂直社区软文 / 专访 | 3000-15000 元 | 看媒体影响力和是否原创采编 |
| 社群/公众号联合活动 | 5000-30000 元 | 可带报名表单和客户线索 |
| 大型社区资源包 | 30000-100000+ 元 | 适合已有明确招商目标时 |

**评估重点**：

- 是否能拿到“品牌方/市场/开发者关系”受众，而不是只有普通开发者；
- 能否提供报名名单、加微信线索或会议预约；
- 是否能沉淀可长期引用的采访/案例链接。

### 3.4 内容 SEO / GEO 写手：适合做长期可见度

**能做什么**：

- 写“AI 产品推广怎么做”“开发者营销渠道有哪些”“技术博主报价”等长尾内容；
- 做竞品词、场景词、服务词页面；
- 把案例整理成可被搜索和 AI 问答引用的内容资产；
- 给官网补 FAQ、案例、报价解释、行业页。

**预算参考**：

| 交付 | 参考价格 |
|---|---:|
| 单篇 SEO/GEO 长文 | 500-3000 元 |
| 10 篇专题内容包 | 5000-20000 元 |
| 官网转化文案 / 服务页重写 | 3000-15000 元 |
| 月度 SEO/GEO 顾问 | 5000-30000 元/月 |

**评估重点**：

- 是否懂开发者产品，不要写成泛营销鸡汤；
- 能否围绕“客户会搜索的问题”写，而不是围绕平台自夸写；
- 是否提供关键词、内链、标题、FAQ、结构化摘要；
- 1-3 个月看收录和品牌词增长，3-6 个月看自然询盘。

### 3.5 社群主 / 私域渠道：适合做小预算快测

**能做什么**：

- 在 AI 产品群、独立开发者群、程序员副业群、DevRel 群里发合作招募；
- 做一次主题分享；
- 组织“AI 产品免费诊断/推广方案诊断”活动；
- 收集意向品牌方名单。

**预算参考**：

| 形式 | 参考价格 |
|---|---:|
| 单群转发 | 100-500 元 |
| 多群矩阵转发 | 1000-5000 元 |
| 主题分享 / 直播 | 1000-10000 元 |
| 按有效线索 | 100-500 元/条 |

**评估重点**：

- 群成员是否有产品方、创始人、运营、市场；
- 是否允许商业信息；
- 群主是否愿意背书，而不是冷冰冰转发；
- 能否用专属二维码/UTM 追踪来源。

### 3.6 PR / 品牌顾问：适合做包装，但不要过早重投

**能做什么**：

- 梳理品牌故事、媒体话术、对外介绍 PPT；
- 对接媒体报道；
- 设计发布节奏；
- 帮助从“个人资源”包装成“开发者营销基础设施”。

**预算参考**：

| 类型 | 参考价格 |
|---|---:|
| 品牌定位/话术咨询 | 5000-30000 元 |
| 单次 PR 发稿包 | 5000-30000 元 |
| 月度 PR 顾问 | 15000-80000 元/月 |

**评估重点**：

- 早期不要买“通稿铺量”，除非能带来真实目标客户；
- PR 更适合在有 3-5 个可讲案例后放大；
- 优先做可沉淀的专访/案例，而不是廉价新闻稿。

## 四、可执行推广打法：建议先做 4 周验证

### 4.1 推荐 MVP：1-2 万元，验证询盘质量

**目标**：用最小预算验证“哪些渠道能带来品牌方询盘”。

| 动作 | 预算 | 交付 | 指标 |
|---|---:|---|---|
| 官网新增 2-3 个案例/服务页 | 0-5000 | 服务页、案例页、FAQ | 页面访问、停留、加微信 |
| 3-5 位技术博主发平台介绍/案例 | 3000-15000 | 图文/短帖/社群转发 | 咨询数、收藏、评论 |
| 5-10 个 AI/开发者社群定向转发 | 1000-5000 | 群转发/群分享 | 有效线索数 |
| 1 场线上分享/诊断活动 | 1000-8000 | 直播/腾讯会议/飞书表单 | 报名数、会后咨询 |

**4 周达标线**：

- 有效品牌方询盘：10-30 条；
- 明确预算客户：3-8 个；
- 试单成交：1-3 单；
- 有效询盘成本：300-1500 元；
- 首单 CAC 不超过首单毛利的 50%。

如果 1-2 万元花完只带来“博主报名”而不是“品牌方询盘”，说明渠道打偏了。

### 4.2 标准版：3-5 万元，做一次正式小战役

**目标**：形成“品牌认知 + 线索 + 案例”三件套。

| 模块 | 建议预算 | 说明 |
|---|---:|---|
| 旗舰内容 | 5000-15000 | 1 篇主理人深度文章 / 平台介绍 / 案例复盘 |
| KOL/KOC 矩阵 | 15000-30000 | 5-10 位技术博主多平台分发 |
| 社群/直播 | 5000-15000 | AI 产品推广诊断会、开发者营销分享 |
| BD 线索 | 5000-15000 | 找 1-2 个 BD 按线索/会议结算 |
| 数据追踪 | 0-5000 | UTM、表单、CRM、复盘报告 |

**达标线**：

- 有效询盘：30-80 条；
- 有预算客户：10-20 个；
- 试单成交：3-8 单；
- 总成交额：5-15 万；
- 战役毛利覆盖推广成本；
- 沉淀 2-3 个可公开或半公开案例。

### 4.3 放大版：10 万元以上，前提是已有转化模型

只有在标准版验证出稳定路径后，才建议上 10 万以上预算。否则容易变成“花钱买热闹”。

可放大的方向：

1. **行业专题**：如“AI 编程工具开发者营销专题”“云服务开发者增长专题”；
2. **年度榜单/白皮书**：发布《AI 产品开发者营销渠道报告》；
3. **长期联运**：和 AI 工具、云服务、课程产品做分佣；
4. **线下活动**：小型闭门会，邀请产品方、博主、DevRel；
5. **海外/出海专题**：面向出海开发者工具做中文市场进入服务。

放大阶段的 KPI 不再只是询盘，而要看：

- 月度新增商机金额；
- 成交周期；
- 客单价；
- 毛利率；
- 复购率；
- 博主交付稳定性；
- 内容资产带来的自然询盘占比。

## 五、钱怎么花：预算优先级排序

### 5.1 最值得花的钱

1. **案例内容**：能解释“谁用了、怎么用、结果如何”；
2. **中腰部技术 KOL 背书**：比泛粉大号更精准；
3. **目标客户社群 BD**：直接接近 AI 产品方/开发者工具团队；
4. **服务页和 FAQ 优化**：让来的人看懂怎么买；
5. **数据追踪系统**：否则不知道钱花在哪有效。

### 5.2 暂时不建议重投的钱

| 不建议重投 | 原因 |
|---|---|
| 泛信息流广告 | 技术产品推广平台需要信任解释，冷广告转化通常低 |
| 纯通稿铺量 | 看起来有声量，但不一定有目标客户 |
| 低价群发 | 可能吸引大量无预算咨询或低质量博主报名 |
| 刷量/刷互动 | 损害平台信用，与“可信技术博主联盟”定位冲突 |
| 大额品牌片/视觉包装 | 早期更缺案例和线索，不缺好看的包装 |

## 六、对外可以包装成 4 个获客产品

为了方便别人帮你推广，博主联盟自己也要把“销售入口”标准化。建议对外先放 4 个产品：

| 产品 | 目标客户 | 起步价 | 核心交付 |
|---|---|---:|---|
| AI 产品推广诊断 | 不知道怎么投的早期产品 | 0-999 元引流 / 可免费限量 | 30 分钟诊断 + 推广建议 |
| 技术博主试投包 | 想小预算测试的品牌方 | 3000-10000 元 | 2-5 位博主内容/转发 + 数据回收 |
| 开发者内容增长包 | 有月度推广需求的产品 | 15000-50000 元/月 | 选题、博主撮合、发布、复盘 |
| GEO 内容资产包 | 重视搜索/AI 搜索的品牌 | 10000-50000 元/月 | 场景词/竞品词/教程/案例内容 |

**注意**：推广时不要只说“我们有 41 位博主”，而要说“你这个产品适合哪几类博主、怎么发、预期拿到什么资产、怎么复盘”。

## 七、怎么评估推广人是否靠谱

找人推广之前，先给对方一张评分表。

| 维度 | 权重 | 高分表现 |
|---|---:|---|
| 目标客户匹配 | 25% | 能触达 AI 产品方、DevTools、云厂商、DevRel，而不是泛流量 |
| 技术理解 | 20% | 能讲清楚开发者营销，不把技术产品当普通消费品卖 |
| 可追踪交付 | 20% | 愿意用 UTM、表单、专属二维码、CRM 记录线索 |
| 过往案例 | 15% | 做过 B2B / SaaS / 技术社区 / 开发者生态推广 |
| 结算合理 | 10% | 能接受基础费 + 结果或按有效线索/会议结算 |
| 合规边界 | 10% | 不刷量、不虚假承诺、不乱发敏感渠道 |

低于 70 分不建议合作；70-85 分可以小预算试单；85 分以上才值得给连续预算或分成。

### 7.1 合同/口头约定要写清楚

1. 什么叫有效线索；
2. 是否允许重复线索；
3. 客户保护期多久；
4. 付款节点；
5. 是否包含内容制作；
6. 是否允许使用博主联盟 logo / 案例；
7. 不得承诺平台无法控制的转化；
8. 不得使用刷量、垃圾群发、违规导流。

## 八、数据看板：至少追这 12 个指标

| 阶段 | 指标 |
|---|---|
| 流量 | 官网 UV、服务页 UV、案例页 UV、来源渠道 |
| 互动 | 加微信数、表单数、资料下载数、活动报名数 |
| 线索 | 有效询盘数、明确预算询盘数、会议数 |
| 销售 | 报价数、成交数、成交额、成交周期 |
| 经济性 | 有效询盘成本、成交 CAC、项目毛利率 |
| 资产 | 新增案例数、搜索收录数、品牌词搜索量 |

最小数据工具不需要复杂：

- 每个渠道一个 UTM 链接；
- 每个合作人一个专属二维码 / 备注口令；
- 统一用飞书表格或 Notion CRM 记录；
- 每周复盘一次：来源、线索质量、成交进展、下一步动作。

## 九、推荐执行路线

### 第 0 周：先补齐转化承接

1. 官网首页首屏明确一句话：帮 AI / 开发者产品找到可信技术博主做推广；
2. 服务页明确 4 个入口：推文、引流、社群、长期联运；
3. 案例页至少展示 3 类案例：AI 产品、开发工具、云服务/技术社区；
4. 准备一页 PDF：博主资源、服务包、报价区间、合作流程、数据样例；
5. 准备表单：产品名、官网、推广目标、预算、期望时间。

### 第 1-2 周：小预算找信号

- 找 3-5 位中腰部技术博主发“博主联盟能解决什么问题”；
- 找 5-10 个 AI 产品/独立开发者/DevTools 社群做定向转发；
- 做一次免费“AI 产品推广诊断”直播或腾讯会议；
- 每个渠道都用不同追踪口令。

### 第 3-4 周：把有效渠道变成试单

- 对有预算线索给出 3 档方案；
- 让愿意试的小客户走 3000-10000 元试投包；
- 每个试投项目都产出复盘截图和案例摘要；
- 用试投结果反向更新官网和销售材料。

### 第 2 个月：放大一个最有效渠道

如果发现“社群 BD 最有效”，就招 1-2 个兼职 BD；如果发现“KOL 内容最有效”，就做月度 KOL 矩阵；如果发现“搜索询盘质量高”，就加 SEO/GEO 内容。

不要同时放大所有渠道。

## 十、最终建议

**一句话定性**：博主联盟当前要推广的不是一个泛平台，而是一个“帮 AI / 开发者产品降低技术博主合作成本”的 B2B 服务。最适合的推广方式是**可信内容 + 垂直社群 + 定向 BD + 案例复盘**，而不是泛广告投流。

**预算建议**：

- **最低验证**：3000-5000 元，找 2-3 个博主 + 若干社群，测话术；
- **推荐验证**：10000-20000 元，跑 4 周，目标 10-30 条有效询盘；
- **正式战役**：30000-50000 元，做 KOL 矩阵 + 社群活动 + BD 线索，目标 3-8 个试单；
- **放大预算**：100000 元以上，必须建立在已知 CAC、成交率、毛利率成立之后。

**最重要的判断标准**：

> 如果一个推广人只能带来阅读量，不一定值得；如果能带来“有官网、有预算、有推广目标”的品牌方询盘，就值得继续合作。

**下一步行动**：

1. 做一份《博主联盟品牌方合作说明书》PDF；
2. 建一个“AI 产品推广免费诊断”表单；
3. 选 5 位技术博主做第一轮背书内容；
4. 找 2 个兼职 BD，按有效会议或成交佣金试跑；
5. 4 周后只保留 CAC 最低、线索质量最高的 1-2 个渠道。

## 一、先做三个关键判断，再谈渠道

绝大多数早期撮合型平台死在“一开始就想做两边”。Claude Code（Opus 4.7）的视角是：在确定渠道之前，必须先回答清楚三个判断题，否则后面所有渠道动作都会被反向放大成噪声。

### 判断 1：当前阶段"真正稀缺"的是哪一边？

博主联盟 v0.1 的实际情况是：

- **博主侧**：主理人是技术作者，有人脉，加博主进群比成交一个客户容易得多。
- **品牌侧**：能付费、能复盘、不打扰博主信用的真实客户才是稀缺品。

结论：**早期把 70% 的获客资源压在需求侧，博主侧只做"够交付当下项目"的最小化运营**。等需求侧成交密度起来，再反向把博主侧从"够用"做到"挑选"。

> **不常见洞见 ①**：双边平台的"双边同步增长"是终局，不是起手式。起手式永远是单边发力——哪边稀缺，哪边发力。博主联盟的稀缺侧是付费意愿明确的品牌方，不是博主。

### 判断 2：客户是"被搜索到的"还是"被推荐到的"？

技术品牌方采买开发者营销服务时，通常走两条路径：

| 路径 | 触发场景 | 决策周期 | 适合的获客方式 |
|---|---|---|---|
| 主动搜索 | 已经被同事 / 上级要求"做开发者推广" | 中短 | SEO / GEO / 案例库 / 落地页 |
| 被动推荐 | 同行、朋友、社区里被安利 | 长 | 案例传播 / 主理人 IP / 行业活动 |

博主联盟当前更适合**双线并行，但主理人 IP + 案例传播是主线，SEO/GEO 是补线**。原因是：

- 早期没有足够多关键词数据支撑 SEO 漏斗；
- AI 产品方更信"谁做过、效果怎么样"，而不是"百度搜出来的"；
- GEO（AI 搜索引用）需要持续高质量内容，正好是博主联盟自身能力辐射的方向。

### 判断 3：什么动作必须自己做，什么动作可以外包？

撮合型平台的护城河本质是 **"判断力 + 关系网 + 案例库"**，这三样都不能外包。其余环节按"是否可 SOP 化"分类：

| 环节 | 能否外包 | 理由 |
|---|---|---|
| 客户线索初筛 | ⚠️ 部分可 | 简单资质筛查可外包，意图判断必须自己来 |
| 销售跟进与报价 | ❌ 不可 | 早期价格不透明，外包人很难谈出毛利 |
| 内容生产（自有渠道） | ✅ 可 | 写稿、视频剪辑、海报设计可外包，选题必须自己定 |
| 投放执行（信息流 / 广点通） | ✅ 可 | 但 v0.1 阶段不建议买量 |
| 案例报告制作 | ⚠️ 部分可 | 数据汇总可外包，复盘判断必须自己做 |
| 博主对接与撮合 | ❌ 不可 | 这是平台核心动作，外包等于自废武功 |
| 客户成功与续约 | ❌ 不可 | 续约取决于信任，信任无法外包 |

> **不常见洞见 ②**：早期外包最危险的不是"做不好"，而是"做完之后没有沉淀"。任何外包动作都必须留下可复用的资产（话术、模板、素材库），否则就是把钱烧成空气。

## 二、需求侧获客：六个渠道的优先级排序

把所有可能渠道列出来排个优先级，避免"看起来都重要、其实都没做透"。

### 2.1 渠道优先级矩阵

| 渠道 | 启动成本 | 起效周期 | 单客成本预估 | 是否可复盘 | 优先级 |
|---|---|---|---|---|---|
| 主理人 IP（掘金 / 知乎 / 公众号 / X） | 低（已有积累） | 即时 | 极低 | 中 | ⭐⭐⭐⭐⭐ |
| 案例传播（客户成功故事） | 中 | 1-2 个月 | 低 | 高 | ⭐⭐⭐⭐⭐ |
| GEO / SEO（围绕"开发者营销"长尾词） | 中 | 3-6 个月 | 低 | 高 | ⭐⭐⭐⭐ |
| 行业活动 / Meetup / 闭门会 | 高 | 即时 | 中 | 中 | ⭐⭐⭐ |
| 冷启动外联（直接 BD 出海工具 / AI 厂商） | 中 | 2-4 周 | 中 | 高 | ⭐⭐⭐ |
| 付费投放（朋友圈 / 信息流 / SEM） | 高 | 即时 | 高 | 低 | ⭐ |

**Claude Code（Opus 4.7）的优先级判断**：前三个渠道（主理人 IP、案例传播、GEO/SEO）应该吃掉至少 70% 的精力，因为它们三者会**互相增强**：

- IP 出内容 → 内容里嵌案例 → 案例形成 GEO 资产 → AI 搜索带回新客户 → 新客户成案例。

这是一个自我加强的飞轮，而付费投放和冷外联是"一次性烧钱"，不进飞轮。

### 2.2 主理人 IP：把"作者"变成"销售"

主理人本身是技术博主，这是博主联盟最被低估的获客通道。

**可执行动作**：

- 每月固定 4-6 篇深度内容，主题围绕"开发者营销方法论 / 博主侧观察 / 案例复盘"；
- 每篇内容末尾留**轻钩子**（不是硬广），如"如果你是 AI 产品方，可以邮件 / 微信加我聊一次"；
- 知乎、掘金、X、公众号四个阵地交叉发布，但每个平台调整语气和切入点；
- 每季度做一次**直播 / AMA**，主题是"AI 产品出海怎么找开发者博主"或"开发者营销的 5 个坑"。

**为什么不发抖音 / 小红书**：博主联盟的目标客户（AI 产品方决策者）不在这两个平台。把精力放在他们真正消费技术内容的地方。

### 2.3 案例传播：把每个项目当未来获客资产做

每个交付完的项目应该至少沉淀：

1. **公开版案例报告**（数据脱敏、客户授权后发布）；
2. **半公开版**（用于销售时一对一发送，含更详细数据）；
3. **内部复盘文档**（团队学习用，含失败和坑）。

**最关键的动作**：与客户签约时就把"案例使用授权"写进合同。等项目结束再去谈授权，成功率会断崖式下降。

### 2.4 GEO / SEO：押注 AI 搜索时代的"被引用率"

传统 SEO 仍要做（sitemap、TDK、H 标签、内链），但更重要的是 **GEO（生成式搜索优化）**——让 AI 在被问到相关问题时引用你的内容。

**核心动作**：

- 围绕"开发者营销 / 技术博主合作 / AI 产品推广 / DevRel 外包"这些长尾词，每个词做 1 篇结构化深度文章；
- 文章必须有**明确的结论 + 可引用的表格 + 数字化数据**，AI 模型更倾向于引用结构化内容；
- 提交到 Bing Webmaster（GPT 系优先抓 Bing 索引）和 Google Search Console；
- 在掘金 / 知乎 / 公众号同步发布，借助这些平台的高权重加速被收录。

> **不常见洞见 ③**：在 AI 搜索时代，"被一次 AI 引用"的价值远高于"被一万次浏览"，因为 AI 引用会持续给你带来后续相关问询。这是个人 / 早期团队最被低估的杠杆。

### 2.5 冷启动外联：精准但不可规模化

冷外联适合"已知道想要哪类客户但还没建立联系"的场景。

**该做的**：

- 列 20-50 家 AI 产品 / 开发者工具 / 出海 SaaS 的目标清单；
- 找到决策人（通常是市场负责人 / DevRel 负责人 / 增长负责人）；
- 发送**带具体观察 + 具体建议**的开场，不发模板。

**不该做的**：

- 群发邮件 / 群发微信；
- 不做调研直接喊"我们有 XX 博主资源"；
- 不留可验证的案例链接。

### 2.6 付费投放：v0.1 不建议碰

原因：

- 客单价不够高（早期 1 个项目可能只有几万元），CAC 一旦超过万级就负毛利；
- 决策周期长，归因困难；
- 投放素材需要大量测试，早期没人没钱测；
- 钱烧完只剩账单，不留资产。

**例外**：如果某次品牌活动 / 案例发布需要短期放大，可以做一次性的小规模朋友圈或 X 推广，但要预算上限（如 5000 元封顶）。

## 三、供给侧（博主）运营：够用即可，不抢资源

需求侧没起量之前，博主侧的目标是"够交付当下项目 + 维护好关系不流失"，不需要追求博主数量。

### 3.1 博主分层

| 层级 | 标准 | 数量目标 | 维护方式 |
|---|---|---|---|
| 核心层 | 准时交付、内容质量稳定、愿意合作 | 10-15 人 | 主理人亲自维护，季度沟通 |
| 协作层 | 合作过 1-2 次，质量可接受 | 30-50 人 | 项目机会触达，发月度精选 |
| 储备层 | 加了联系方式但未合作 | 100-200 人 | 群通知 + 标签化数据库 |

### 3.2 博主侧不该做的事

- 不要为了"博主库数字好看"加大量低质账号；
- 不要把核心博主的报价压到低于市场价，长期会伤信任；
- 不要让博主感到自己只是"流量工具"，要让他们看到合作带来的个人品牌增长。

### 3.3 博主侧的轻量化获客

- 主理人本身在技术圈的关系是最大资产；
- 每季度发一次**博主联盟内部内容榜单**（哪些博主的内容效果最好），既是表彰也是品牌；
- 通过已合作博主推荐新博主，给予轻度激励（比如优先权 / 案例分成）。

## 四、预算分配模型：一个可套用的公式

以下假设博主联盟 v0.1 单月可投入的获客预算为 **3-5 万元**（包含人力时间折算）。

### 4.1 推荐分配比例

| 项目 | 占比 | 月度预算（按 4 万元基准） | 说明 |
|---|---|---|---|
| 主理人内容生产时间 | 30% | 12,000 | 折算主理人 30-40 小时/月 |
| 外包写作 / 设计 / 剪辑 | 15% | 6,000 | 仅外包"可 SOP 化"环节 |
| 案例制作（含设计 / 排版） | 10% | 4,000 | 每月至少出 1 个完整案例 |
| GEO / SEO 工具与素材 | 10% | 4,000 | 包含 SaaS、关键词工具、AI 工具 |
| 冷外联 / BD 时间 | 15% | 6,000 | 折算 BD 人 30 小时/月 |
| 行业活动 / 闭门会 | 15% | 6,000 | 季度峰值，单月可挪用 |
| 实验性付费投放 | 5% | 2,000 | 仅用于放大单点高潜内容 |
| **合计** | 100% | 40,000 | |

### 4.2 预算调整规则

- 如果某渠道连续 2 个月没产生可归因线索，预算砍半；
- 如果某渠道单月线索成本低于均值 50%，预算翻倍；
- 行业活动按季度看而不是按月看，单月超支允许；
- 付费投放必须有"刹车机制"，每周复盘一次。

### 4.3 不应该花钱的地方

- 不应该一开始就做 CRM / 数据看板 SaaS 订阅（用飞书表格 + Notion 足够）；
- 不应该买"博主资源包"或"流量包"，质量无法控制；
- 不应该过早做品牌设计大改版（先靠内容立信，VI 之后再说）；
- 不应该投硬广 / 软文营销（与开发者人群天然冲突）。

## 五、外包决策：边界比"找谁"更重要

### 5.1 外包决策树

判断一个动作是否可以外包，过三道题：

1. **这个动作是否有可被复用的 SOP？** 没有 → 自己做。
2. **这个动作的产出物是否需要主理人判断力？** 是 → 自己做。
3. **这个动作的失误是否会损伤博主 / 客户信任？** 是 → 自己做。

三道题都不满足，才可以外包。

### 5.2 推荐外包项

| 外包项 | 推荐合作形式 | 预算参考（月） |
|---|---|---|
| 公众号 / 知乎排版 | 兼职大学生 / 小工作室 | 1,500-3,000 |
| 视频剪辑（如果做视频） | 兼职剪辑师 | 2,000-5,000 |
| 案例报告排版 / Figma 设计 | 兼职设计师 | 2,000-4,000 |
| 关键词调研 / SEO 数据 | 工具 + 短期外包 | 1,000-2,000 |
| 客户线索初筛（资质核验） | 实习生 / 兼职 | 2,000-3,000 |

### 5.3 不推荐外包项

- 销售谈判（价格透明度不够，谈不出毛利）；
- 博主对接（关系网无法转移）；
- 选题判断（错一篇内容损伤一年信任）；
- 案例复盘判断（数据解读会失真）；
- 客户成功 / 续约（信任不能交接）。

### 5.4 外包人才挑选标准

- 优先找**已在行业内的兼职博主 / 创作者**，而不是泛设计 / 写作外包；
- 用"小任务付费试合作"代替"长期合同"，先做 1-2 单看产出再续；
- 必须留有产出物的源文件 / 编辑权限，避免人走资产走。

## 六、评估模型：用 5 个核心指标管理增长

不要试图盯 30 个指标，盯不过来。Claude Code（Opus 4.7）推荐用 5 个核心指标做月度复盘，其他指标只作为参考。

### 6.1 五个核心指标

| 指标 | 公式 | 健康区间（v0.1） | 用途 |
|---|---|---|---|
| **MQL 数**（月度合格线索） | 主动询价 + 主动加微信咨询 | ≥ 15 / 月 | 衡量获客漏斗顶端 |
| **MQL → 成交转化率** | 成交客户数 / MQL 数 | 15%-25% | 衡量销售判断与匹配度 |
| **平均客单价** | 月度营收 / 成交客户数 | 2-8 万元 | 衡量服务包定价 |
| **客户复购率**（半年） | 半年内复购客户数 / 半年内总客户数 | ≥ 30% | 衡量交付质量与续约能力 |
| **单线索内容成本** | 月度获客投入 / MQL 数 | ≤ 1,500 元 | 衡量渠道效率 |

### 6.2 二级指标（参考用）

- 内容月度发布数（绝对值，不必追求最大化）；
- 内容平均阅读 / 互动；
- GEO 引用次数（在 Kimi / GPT / DeepSeek 等被引用的次数，手动抽查）；
- 博主活跃度（30 天内有过沟通的博主比例）；
- 项目准时交付率；
- NPS（每个项目结束后简单问一次）。

### 6.3 月度复盘 SOP

每月最后一周做一次复盘，固定 4 个问题：

1. **本月哪一个渠道贡献了最多 MQL？为什么？**
2. **本月有没有出现"超预期高效"或"超预期低效"的动作？**
3. **下个月预算需要调整哪些项？**
4. **博主侧 / 客户侧出现了哪些"非线性变化"（如某博主突然走红、某客户突然爆单）？**

复盘文档归档，半年回头看一次趋势。

### 6.4 不应该看的指标

- **博主注册总数**（虚荣指标，不代表生产力）；
- **公众号关注数**（除非有具体转化路径，否则只是数字）；
- **官网 UV**（早期太少，没统计意义）；
- **媒体曝光次数**（除非直接关联到 MQL）。

> **不常见洞见 ④**：早期最容易自欺欺人的指标是"内容阅读量"。一篇 10 万+ 的文章如果带不来 1 个 MQL，价值低于一篇 1000 阅读但带来 3 个 MQL 的文章。**别把流量当增长，别把曝光当转化。**

## 七、90 天落地行动清单

把上面所有判断浓缩为一个可执行的 90 天计划。

### Day 1-30：建底子

- [ ] 把现有所有合作过的客户整理成案例草稿（即使没正式发布，先内部沉淀）；
- [ ] 主理人定下每月固定 4 篇内容的选题节奏，开始第一批发布；
- [ ] 建立博主三层档案表（Notion / 飞书），核心层 10 人先标好；
- [ ] 列出 20 家目标客户，开始第一轮冷外联（每周 5 家）；
- [ ] 上线官网 SEO 基础（sitemap、TDK、H 结构、内链）；
- [ ] 建立月度数据复盘模板。

### Day 31-60：跑漏斗

- [ ] 发布至少 1 个公开版案例报告；
- [ ] 选 5 个长尾关键词，每个写一篇 GEO 文章；
- [ ] 跑通"询价 → 报价 → 合同 → 启动 → 交付 → 复盘"完整 SOP；
- [ ] 开始外包排版 / 设计，释放主理人精力；
- [ ] 第一次月度复盘，明确哪些渠道值得加注。

### Day 61-90：找飞轮

- [ ] 主理人内容里嵌入案例引用，形成"内容 → 案例 → 内容"循环；
- [ ] 邀请核心博主参与一次小规模直播 / AMA，借势放大品牌；
- [ ] 出第一份"博主联盟季度内容榜单"，对内对外双重价值；
- [ ] 完整复盘 90 天五个核心指标，决定下季度预算调整方向；
- [ ] 如果 MQL 已稳定 ≥ 15/月，可启动"博主侧分层升级"。

## 八、常见陷阱与红线

### 陷阱 1：双边同时发力，结果两边都不饱和

撮合型平台早期不能"两边都重"，必须押单边。博主联盟当下押需求侧。

### 陷阱 2：把"博主多"当卖点，忽视"博主精"

向品牌方说"我们有 500 个博主"远不如说"我们有 30 个 AI 方向核心博主，全部签了优先合作"。**质量信号永远比数量信号有说服力。**

### 陷阱 3：用流量指标管理业务

阅读量、UV、关注数都不是收入。盯住 MQL、转化率、客单价、复购率，剩下都是辅助。

### 陷阱 4：外包变成"花钱买噪声"

外包必须留下可复用资产。如果一个外包项做完三次仍然每次都要重新指导，要么换人，要么收回自己做。

### 陷阱 5：主理人 IP 与公司品牌不分

短期主理人 IP 会带来获客，但长期必须沉淀公司层面的资产（案例库、博主库、SOP），否则主理人成为单点故障。

### 红线：不能为短期收入伤博主信用

灰产产品、夸大宣传产品、不合规产品哪怕给出高价也要拒。**博主信用一旦透支，整个网络的供给侧就崩了**，这是不可逆损失。

## 九、与默认版本的差异说明

如果你也读了同一主题的默认（Cursor）版本，本 Claude Code（Opus 4.7）版本的差异主要在：

- **更早收敛到决策点**：先用"三个判断"框定方向，再展开渠道；
- **更强的反直觉判断**：对"双边同时发力""博主多即好""阅读量等于增长"等常见错觉做了直接反驳；
- **更明确的红线和不该做的事**：很多增长文章只讲"应该做什么"，本文等量列出"不应该做什么"；
- **更具体的预算公式与可套用区间**：不止给方向，还给可对照的数字。

如果你需要更线性、更接近"操作手册"的版本，可在文末切换器切到默认版本对比阅读。

## 十、结论

博主联盟 v0.1 阶段自身的获客推广，本质是回答三个问题：

1. **谁是当下最稀缺的一边？** → 需求侧（付费品牌方）。
2. **靠什么飞轮自我增强？** → 主理人 IP × 案例传播 × GEO 三角。
3. **怎么知道飞轮转起来了？** → 五个核心指标的稳定上行。

不要把早期获客做成"广撒网"，做成"押单边 + 飞轮自增强 + 严控外包边界 + 数据复盘可解释"。

把每月固定 3-5 万元预算押在能复盘的渠道上，把每个项目沉淀为下一次销售资产，把每个博主当成可长期合作的伙伴而不是耗材。这套打法不性感，但能让 v0.1 在 90-180 天里跑出可复用的获客系统，而不是烧完一波预算后再次回到原点。
